# RemoveThingTopo {#reference_ocy_trz_wdb .reference}

You can call this operation to remove the topology between a specified gateway and sub-device.

## Request parameters {#section_aym_wtt_xdb .section}

|Name|Type|Required|Description|
|:---|:---|:-------|:----------|
|Action|String|Yes|The operation that you want to perform. Set the value to RemoveThingTopo.|
|IotId|String|No| The identifier of the device that you want to delete.

 **Note:** If you use this parameter, ProductKey and DeviceName are not required. IotId is a globally unique identifier \(GUID\), and corresponds to a combination of ProductKey and DeviceName. If you use both IotId and the combination of ProductKey and DeviceName, the system follows IotId.

 |
|ProductKey|String|No| The product key of the device that you want to delete.

 **Note:** If you use this parameter, DeviceName is required.

 |
|DeviceName|String|No| The name of the device that you want to delete.

 **Note:** If you use this parameter, ProductKey is required.

 |

## Response parameters {#section_dxh_m5t_xdb .section}

|Name|Type|Description|
|:---|:---|:----------|
|RequestId|String|The GUID generated by Alibaba Cloud for the request.|
|Success|Boolean|Indicates whether the call is successful. A value of true indicates that the call is successful. A value of false indicates that the call has failed.|
|ErrorMessage|String|The error message returned when the call fails.|
|Data|Boolean|The result of removing the topology. A value of true indicates that the deletion is successful, and a value of false indicates that the deletion has failed.|
